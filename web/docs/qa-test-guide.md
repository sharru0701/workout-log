## 1. 사전 준비

### 1-1. 실행 환경

```bash
pnpm run dev:check
pnpm install
pnpm db:migrate
pnpm db:seed
pnpm dev
```

검증 URL: `http://localhost:3000`

권장 브라우저/디바이스:
- Desktop: Chrome 최신, Safari 또는 Firefox 중 1개
- Mobile viewport: iPhone 14 Pro(390×844), Android Pixel 계열

### 1-2. 테스트 데이터 원칙

- 인증 유저: 서버 환경변수 `WORKOUT_AUTH_USER_ID` (미지정 시 `dev`)
- 테스트 전후 데이터 보존 필요 시 `설정 > 데이터 내보내기` 에서 JSON 백업

### 1-3. 이슈 심각도

| 등급 | 기준 |
|------|------|
| S1 (치명) | 저장 불가, 데이터 유실, 페이지 진입 불가 |
| S2 (높음) | 핵심 플로우는 가능하나 우회 필요, 잘못된 데이터 저장 |
| S3 (중간) | UX 혼란/문구/상태 표시 문제, 재시도 시 해결 가능 |
| S4 (낮음) | 미세 UI 이슈, 가독성/정렬/카피 품질 |

## 2. 라이트 유저 E2E 테스트

**유저 정의**: 빠르게 오늘 운동 기록/저장 완료. 복잡한 설정 최소화, 기본 모드 선호.
**성공 정의**: 플랜 준비 → 세션 생성/적용 → 운동 추가(옵션) → 저장까지 5분 내 완료

### L-01. 홈 첫 진입 인지성

| 항목 | 내용 |
|------|------|
| 목적 | 핵심 CTA 3개가 상단에서 즉시 보이는지 확인 |
| 절차 | `/` 진입 → `빠른 시작` 섹션 확인 → `고급 도구` 분리 여부 확인 |
| 기대 | CTA 3개 상단 노출, `권장 순서: 프로그램 준비 후 오늘 운동 시작.` 안내 표시 |
| 실패 | CTA 누락 또는 고급 기능이 과도하게 먼저 노출 |

### L-02. 플랜 빠른 생성 (단일)

| 항목 | 내용 |
|------|------|
| 목적 | 최소 입력으로 플랜 생성 가능 여부 |
| 절차 | 홈 `커스텀 프로그램 만들기` → `/plans/manage?create=1&type=SINGLE` → 바텀시트 자동 오픈 확인 → 템플릿/버전 선택 후 `Create SINGLE Plan` |
| 기대 | 생성 성공 안내 + 플랜 카드 목록 상단에 신규 플랜 표시 |
| 실패 | 시트 미오픈, 버전 선택 불가, 생성 후 목록 갱신 실패 |

### L-03. 플랜 선택 후 오늘 운동 진입

| 항목 | 내용 |
|------|------|
| 절차 | `/plans/manage` 에서 플랜 `선택` → 카드의 `오늘 운동` 클릭 |
| 기대 | `planId`, `date`, `autoGenerate=1` 파라미터 포함해서 이동, 선택 플랜이 상단 플랜 셀렉트에 반영 |
| 실패 | 다른 플랜으로 진입, planId 누락 |

### L-04. 기본 모드 세션 생성/적용

| 항목 | 내용 |
|------|------|
| 목적 | 라이트 유저 핵심 1순위 플로우 |
| 절차 | `/workout/today/log` 기본 모드 확인 → `1) 세션 생성/적용` 클릭 |
| 기대 | `계획 세트 N개를 불러와 적용했습니다 (...)` + 세트 목록 카드 표시 + 저장 버튼 활성화 |
| 실패 | 생성 버튼 무반응, 성공 메시지 없이 세트 미생성 |

### L-05. + 운동 추가

| 항목 | 내용 |
|------|------|
| 절차 | `2) + 운동 추가` → 추천 운동 탭 → 검색어 입력 후 `직접 추가` 실행 |
| 기대 | 성공 메시지 + 동일 운동명 추가 시 `setNumber` 자동 증가 + 리스트 즉시 반영 |
| 실패 | 추가 후 리스트 미반영, setNumber 중복/역순 |

### L-06. 세트 입력 UX

| 항목 | 내용 |
|------|------|
| 절차 | 세트 행 `반복/중량/RPE` 수정 → `완료 후 다음` → `이전 복사` → `아래에 삽입` |
| 기대 | 완료 후 다음 세트 자동 추가, 이전 복사, 동일 운동 기준 다음 setNumber 생성 |
| 실패 | 값 입력 후 즉시 원복, 행 추가/복사가 다른 운동으로 오염 |

### L-07. 온라인 저장 + 세션 상세 확인

| 항목 | 내용 |
|------|------|
| 절차 | `3) 운동 기록 저장` → `세션 상세 열기` 링크 |
| 기대 | `로그를 저장했습니다: {logId}` + `/workout/session/{logId}` 진입 + 비교표 표시 |
| 실패 | 저장 성공 메시지/링크 없음, 상세에서 404/빈 데이터 |

### L-08. 빈 상태 구조 검증

| 항목 | 내용 |
|------|------|
| 절차 | 새 플랜/새 날짜로 세트 비어있는 상태 유도 |
| 기대 | 빈 상태 안내 + 해결 액션이 같은 영역에서 제공 |
| 실패 | 빈 상태에서 탈출 동선 부재 |

### L-09. 지난 운동 반복

| 항목 | 내용 |
|------|------|
| 절차 | 저장된 로그 1개 이상 있는 상태에서 `지난 운동 반복` 클릭 |
| 기대 | 직전 로그 세트가 현재 입력 세트로 채워짐, 성공 메시지에 직전 수행 시각 포함 |
| 실패 | 이전 로그 있는데 `이전 운동 기록이 없습니다.` 발생 |

### L-10. 오프라인 저장 및 자동 동기화

| 항목 | 내용 |
|------|------|
| 절차 | DevTools Network `Offline` → 저장 실행 → 온라인 복귀 |
| 기대 | 오프라인 저장 성공 메시지 → `동기화 대기: N` → 자동 동기화 또는 `지금 동기화` 버튼 동작 |
| 실패 | 오프라인 저장 자체 실패, 온라인 복귀 후 무기한 대기 |

### 라이트 유저 합격선

- L-01 ~ L-10 중 S1/S2 이슈 0건
- 생성/추가/저장 E2E 1회 이상 완주
- 오프라인 fallback 1회 성공

## 3. 헤비 유저 E2E 테스트

**유저 정의**: 프로그램 설계, 세션 커스터마이징, 분석/운영 지표 기반 개선.
**성공 정의**: 고급 기능이 많아도 플로우가 분절되지 않고 커스터마이징 결과가 생성/기록/통계까지 연결됨

### H-01. 템플릿 포크

| 절차 | `/templates/manage` → 공개 템플릿 `Fork` → `내 개인 템플릿` 목록 확인 |
|------|---|
| 기대 | `Forked template: ...` + `PRIVATE` 표시 |
| 실패 | 성공 메시지 있는데 목록 미반영, 포크 후 `버전 생성` 비활성 |

### H-02. 템플릿 버전 생성 (MANUAL)

| 절차 | 개인 MANUAL 템플릿 → Session/Item/Set 추가/수정 → changelog 입력 후 `버전 생성` |
|------|---|
| 기대 | `{slug} v{n} 버전을 생성했습니다.` + 버전 셀렉트에 신규 버전 추가 |
| 실패 | 편집 내용 일부만 저장, 버전 번호 증가 누락 |

### H-03. 템플릿 버전 생성 (LOGIC)

| 절차 | 개인 LOGIC 템플릿 → 로직 안전 파라미터 변경 + Exercise substitutions 추가 → 버전 생성 |
|------|---|
| 기대 | 재선택 시 변경한 파라미터/대체 규칙 유지 |
| 실패 | 일부 필드가 기본값으로 돌아감 |

### H-04. 플랜 생성 3종

| 절차 | `/plans/manage` → 플랜 만들기 → SINGLE / COMPOSITE / MANUAL 각 1회 |
|------|---|
| 기대 | 각 타입 생성 성공 메시지 + 카드에 신규 항목 추가 |
| 실패 | 특정 타입만 생성 불가, 생성 후 카드 액션 오작동 |

### H-05. 고급 모드 생성 경로 4종

| 절차 | 고급 모드 전환 → `오늘 기준 생성` / `선택 날짜 생성` / `주차/일차 기준 생성` / `계획 세트 적용` 각각 실행 |
|------|---|
| 기대 | 각 경로 성공/에러 메시지 명확 + 스냅샷 JSON 구조 노출 |
| 실패 | 생성 됐는데 적용 불가, 이전 생성 결과가 섞여 표시 |

### H-06. 세션 오버라이드

| 절차 | `세션 오버라이드` 시트 → `보조 운동 고정` → `운동 교체` |
|------|---|
| 기대 | 성공 메시지 후 재생성 결과에 accessory/replacement 반영 |
| 실패 | 오버라이드 성공인데 결과 미반영 |

### H-07. 캘린더 워크스페이스

| 절차 | `/calendar/manage` → month/week 전환 → 날짜 셀에서 `생성` → `날짜 열기` → `AUTO_GENERATE` 전환 후 재검증 |
|------|---|
| 기대 | 셀 상태 표시 정확 + AUTO_GENERATE 시 Today 진입 시 자동 생성 트리거 |
| 실패 | 캘린더에서 생성한 세션과 Today 날짜 컨텍스트 불일치 |

### H-08. 통계 대시보드 필터/비교

| 절차 | `/stats/dashboard` → 프리셋 `7/30/90일` 전환 → 필터 시트에서 플랜/집계단위/exercise 변경 |
|------|---|
| 기대 | 필터 변경이 요약 카드 + 상세 표에 동기 반영, 로딩/에러 섹션별 분리 |
| 실패 | 요약은 바뀌는데 표는 이전 데이터 유지 |

### H-09. UX 퍼널/요약/비교모드/내보내기

| 절차 | 대시보드 `UX 퍼널` 카드 확인 → `CSV 내보내기` → `비교 모드` ON → `스냅샷 JSON/CSV` 다운로드 |
|------|---|
| 기대 | 퍼널 단계/전환율 표시 + 비교 모드 테이블 표시 + export 파일 정상 |
| 실패 | compare toggle 후 표 비노출, export 4xx/5xx |

### H-10. UX 기준치 반영 검증

| 절차 | `/settings/ux-thresholds` → 글로벌 3개 변경 → 플랜별 기준치 변경 → 대시보드 임계치 카드 확인 → 플랜 오버라이드 해제 |
|------|---|
| 기대 | 저장 성공/롤백 안내 명확 + `plan override → global` 우선순위 동작 |
| 실패 | 설정 화면 저장 성공인데 대시보드 목표치 불일치 |

### H-11. 저장 정책 롤백 시뮬레이션

| 절차 | `/settings/save-policy` → `다음 저장 실패` ON → 토글 변경 → 실패 안내/롤백 확인 → 재시도 |
|------|---|
| 기대 | row 단위 잠금/롤백/경고 문구 표시 → 재시도 시 성공 노출 |
| 실패 | 실패 후 값 롤백 안됨, 전체 화면 잠금 |

### H-12. 데이터 내보내기

| 절차 | `/settings/data` → `JSON 다운로드` → `CSV 다운로드` |
|------|---|
| 기대 | 파일 즉시 다운로드. JSON은 템플릿/플랜/세션/로그 포함, CSV는 workout_set 행 기반 |
| 실패 | 다운로드 무반응, CSV 400 에러 |

### H-13. 운영 마이그레이션 텔레메트리

| 절차 | `/stats/dashboard` 운영 마이그레이션 섹션 → 조회 프리셋 전환 → `문제 상태만` ON/OFF → JSON/CSV 다운로드 |
|------|---|
| 기대 | 필터 전환 시 표/요약 갱신 + critical/warn/ok 배지 일관 표시 + 핵심 통계 로딩과 독립 |
| 실패 | 마이그레이션 섹션이 핵심 통계 로딩을 블로킹 |

### 헤비 유저 합격선

- H-01 ~ H-13 S1/S2 이슈 0건
- 커스터마이징 결과가 생성/저장/통계까지 추적 가능

## 4. 비동기 UX 연속성 점검

**사전 조건**:
- OS `Reduce Motion` 설정 OFF
- 네트워크 Throttling: `Fast 3G` 또는 DevTools `Slow 4G` 로 1회 이상 점검

### Scenario 1: 하단 탭 전환

1. 홈(`/`)에서 `프로그램 스토어` 탭 누름
2. 캡슐 인디케이터가 한 번에 좌우로 이동하는지 확인
3. 전환 중 이중 흔들림 없이 1회 이동으로 끝나는지 확인
4. 전환 직후 `설정 값 없음` 플래시가 없는지 확인

### Scenario 2: Plans / Templates 로드

1. `/plans/manage` 진입 후 초기 로드 동안 빈 상태 문구가 먼저 뜨지 않는지 확인
2. 데이터 로드 완료 후 실제 리스트 렌더 확인
3. `/templates/manage` 동일 검증

### Scenario 3: Stats 지연 쿼리 플로우

1. `/stats/dashboard` 진입 후 코어 지표 로드 중 empty-state 조기 노출 없는지 확인
2. 세부 지표(퍼널/시계열/PR)가 뒤늦게 로드될 때도 섹션별 empty-state 플래시 없는지 확인
3. 필터 변경 후 재조회에서도 empty-state가 조회 완료 후에만 나타나는지 확인

### Scenario 4: 운동 컨텍스트

1. `/workout/today/log` 진입 직후 `플랜 미선택` 경고가 선행 노출되지 않는지 확인
2. `/workout/session/[logId]` 진입 시 공백 프레임 없이 로딩→데이터 렌더로 이어지는지 확인

### 합격 기준

- 로딩 전환 중 empty-state가 먼저 튀는 장면 미관찰
- 동일 동작 반복 3회에서 재현성 있게 깜빡임/이중모션 미발생

### 자동화 테스트

```bash
pnpm --dir web run test:async-ux:continuity
pnpm --dir web run test:async-ux:stats-1rm
```

## 5. UX 흐름 품질 체크리스트

- [ ] 첫 화면에서 다음 액션이 즉시 보이는가
- [ ] 실패 시 복구 동선이 같은 화면에서 제공되는가
- [ ] 로딩/에러 상태가 기능 단위로 분리되어 과도하게 막히지 않는가
- [ ] 성공 피드백 문구가 구체적인가 (무엇이 저장/생성됐는지)
- [ ] 오프라인/온라인 전환 시 사용자가 현재 상태를 이해할 수 있는가
- [ ] 고급 기능이 기본 플로우를 방해하지 않는가

## 6. 테스트 운영안

| 라운드 | 범위 | 예상 시간 |
|--------|------|-----------|
| 1차 | 라이트 케이스 전체 (L-01~L-10) | 40~60분 |
| 2차 | 헤비 케이스 전체 (H-01~H-13) | 90~150분 |
| 3차 | 이슈 수정 후 라이트 핵심 회귀 (L-02, L-04, L-05, L-07, L-10) | 20~30분 |
