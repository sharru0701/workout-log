- **설명**: 설계 이유·근거·한계다. 규칙을 새로 만들지 않는다.

## 1. 한 줄 정의

REF5는 **불규칙한 주 2–4회 일정에서 하이바 스쿼트를 항상 우선하고, 정확히 다섯 종목의 유효 반복 결과를 판정창에 쌓아 직접 작업 기준을 2.5 kg 단위로 조절하는 세션 기반 비실패 스트렝스 프로그램**이다.

REF5는 시중 프로그램을 그대로 구현한 것이 아니다. 고정 제약과 연구가 지지하는 넓은 방향을 결합한 **제약 최적화 후보**이며, 유일하거나 과학적으로 입증된 최적해라고 표현하면 안 된다.

- 의도한 변경이면 이 문서, 코드, 테스트, 버전, replay/migration 정책을 같은 변경에서 갱신한다.

- 이미 동결된 과거 스냅샷은 새 규칙으로 소급 해석하지 않는다.

## 3. 프로그램 순도: 반드시 유지할 경계

### 3.1 포함하는 것

- 종목: `SQ`, `PULL`, `BP`, `DL`, `OHP` 정확히 다섯 개
- SQ: 모든 처방에서 같은 **High-Bar Back Squat**
- PULL: **Weighted Pull-Up**, 체중과 추가중량을 합친 총중량 기준
- 세션 기반 자동 생성, PULL↔BP 집중 큐, SQ 하드 밀도 제한
- 정상/마이크로 세션
- 유효 반복과 종료 사유로 자동 계산한 `PASS/HOLD/FAIL/INVALID`
- 직접 작업 기준, 파생 작업중량, 제어 REF, DL/OHP 상한
- 판정창, 같은 흐름의 2연속 FAIL, 정체 마이크로, replay

### 3.2 포함하지 않는 것

다음은 REF5 v1.2에 추가하면 안 된다.

- 클라이밍 또는 다른 스포츠·외부 활동에 따른 처방 변경
- 로우바 스쿼트, 스쿼트 종목 교체, 선택식 스쿼트 변형
- 폭발·스피드·파워 스쿼트 같은 별도 운동 속성
- 여섯 번째 종목, 선택 보조운동, 종목 추가·교체·생략 기능
- 실측 1RM 테스트, 런타임 1RM/e1RM·Training Max 상태, `%1RM` 기반 진행
- AMRAP, RPE, RIR, 속도 등급 입력
- 의도적 실패, 추가 반복, 성과 테스트 세트
- 주차, 유한 사이클, 블록, 피킹, 고정 디로드 주간
- Asymptote의 TM·사이클·AMRAP·light-block 상태 재사용
- 누락 세션 보충, 다음 세션에 세트 이월, 상한 여유 적립
- 무릎 상태나 통증을 포함한 의료·재활 분기

코드에 `climb`, `climbing` 같은 문자열이 보일 수 있다. 이는 v1.1 입력을 **거부하기 위한 삭제 표식과 회귀 테스트**다. 활성 세션 판단에 사용하거나 새 상태로 복원하면 안 된다.

공통 workout schema의 `rpe: 0` 같은 값도 저장 호환용 고정값일 뿐 REF5 입력이나 판정값이 아니다.

## 4. 핵심 용어

| 용어 | 정확한 뜻 |
|---|---|
| 직접 기준 | 진행 상태가 직접 보유하는 작업중량. `SQ H3`, `BP 집중`, `PULL 집중 총중량`, `DL`, `OHP` |
| 파생 기준 | 직접 기준에서 공식으로 다시 계산하는 `SQ H2`, `SQ V`, `BP 볼륨`, `PULL 볼륨 총중량` |
| 제어 REF | 직접 기준을 추적하기 위한 내부 환산값. 실제·추정 1RM이 아님 |
| 하드 | 정상 세션의 SQ `H3` 또는 `H2`. SQ `V`와 마이크로는 하드가 아님 |
| 집중 | 정상 세션의 PULL 또는 BP `3×3`. 볼륨 처방과 마이크로는 집중이 아님 |
| 볼륨 | SQ `V`, BP `1×5`, PULL `1×6` 처방 |
| 흐름(stream) | 2연속 FAIL을 독립 추적하는 처방 단위 |
| 판정창 | 증량 여부를 판단하기 위해 유효 하드·집중·보조 노출을 모으는 창 |
| 유효 노출 | `PASS`, `HOLD`, `FAIL`. `INVALID`는 제외 |
| 프로그램일 | 실제 SQ 첫 작업세트를 시작한 계획 시간대의 달력일 |
| 완료 세션 | 예정된 모든 종목의 결과를 포함해 세션을 닫은 상태 |

`H3`, `H2`, `V`는 같은 하이바 스쿼트의 **세트·반복·중량 처방명**이다. 기술, 템포, 속도, 폭발성, 종목 변형을 뜻하지 않는다.

## 5. 고정 제약과 시작 상태

### 5.1 고정 제약

- 계획 상태가 보유하는 시작값은 1RM·e1RM·TM이 아니라 kg 단위의 직접 작업 기준이다.
- e1RM은 계획 생성 전 첫 처방 추천에만 사용할 수 있으며 런타임 상태나 이후 진행 입력이 될 수 없다.
- 각 시작값은 `2.5–500 kg` 범위의 `2.5 kg` 격자값이어야 한다.
- PULL 시작값은 추가중량이 아니라 **체중+추가중량의 목표 총중량**이다.
- SQ가 최우선이고 모든 세션에서 첫 번째다.
- SQ·PULL·BP가 주종목, DL·OHP가 보조종목이다.
- `DL 제어 REF ≤ SQ 제어 REF`
- `OHP 제어 REF ≤ BP 제어 REF × 0.5`
- 불규칙한 주 2–4회 수행을 전제로 다음 실제 세션을 자동 생성한다.
- 대회 피킹 없이 지속 진행한다.

### 5.2 최초 직접 기준과 기본값

| 직접 기준 | 기본값 | 처방 |
|---|---:|---:|
| SQ H3 | 82.5 kg | 3×3 |
| BP 집중 | 82.5 kg | 3×3 |
| PULL 집중 | 총중량 87.5 kg | 3×3 |
| DL | 72.5 kg | 2×4 |
| OHP | 32.5 kg | 2×6 |

표 값은 고급 직접 입력 화면의 기본값이며 사용자가 바꿀 수 있다. 앱은 제출값을 정본으로 검증·저장하고 제어 REF와 파생 기준을 다시 계산한다. 클라이언트가 보낸 제어 REF는 신뢰하지 않는다.

- DL 시작값은 입력한 SQ에서 파생된 DL 상한 이하여야 한다.
- OHP 시작값은 입력한 BP에서 파생된 OHP 상한 이하여야 한다.
- 고급 직접 입력에서 조건을 어긴 값은 조용히 낮추지 않고 계획 생성을 거부한다.
- 계획이 생성되면 시작값은 replay의 기저 상태가 되므로 수정할 수 없다. 다른 시작값을 쓰려면 새 계획을 만든다.
- 기존 계획과 이미 동결된 세션에는 새 입력을 소급 적용하지 않는다.

최초 큐는 `PULL`, 최초 SQ 하드 차례는 `H3`다. 시작 도우미를 사용해도 이 상태는 바뀌지 않는다.

### 5.3 일회성 e1RM 시작 도우미

새 계획의 기본 입력 방식은 **최근 기록·e1RM**이다. 이는 실측 최대중량 검사나 TM 기반 프로그램 전환이 아니라, 사용자가 첫 작업중량을 정하기 쉽게 만드는 계획 생성 전 보조 계산이다.

자동 추천 기록의 자격은 다음과 같다.

- 현재 시점 기준 최근 `56일` 안의 기록
- `1–10회`로 완료한 세트
- 정확한 다섯 운동 정체성만 허용: High-Bar/Back Squat, Bench Press, Pull-Up, Conventional Deadlift, Overhead Press
- 로우바·프론트 스쿼트, RDL, 로우, 랫풀다운 등 유사 종목은 사용하지 않음
- PULL 기록중량은 저장된 체중/총중량 메타데이터로 복원한 **총중량**. 신뢰할 총중량 메타가 없는 과거 기록은 자동추천에서 제외
- 각 종목에서 Epley `e1RM = 중량 × (1 + 반복/30)`이 가장 높은 자격 기록 한 개 사용

추천 첫 작업중량은 기존 제어 REF 공식의 역계수를 사용해 계산한다.

```text
SQ H3                  = floor2.5(SQ e1RM × 82.5 / 104)
BP 집중                = floor2.5(BP e1RM × 82.5 / 101)
PULL 집중 목표 총중량 = floor2.5(PULL 총중량 e1RM × 87.5 / 108)
DL                     = floor2.5(DL e1RM × 72.5 / 100)
OHP                    = floor2.5(OHP e1RM × 32.5 / 50)
```

- 다섯 값 중 하나라도 없으면 앱은 개인 기본값으로 몰래 채우지 않는다. 사용자가 알고 있는 e1RM을 입력하거나 고급 직접 입력으로 전환한다.
- 계산 결과의 DL/OHP가 현재 SQ/BP 상한을 넘으면 상한까지 낮추고 조정 전·후 값을 사용자에게 명시한다.
- 계산된 첫 처방을 계획 생성 전에 보여준다.
- 제출 뒤에는 e1RM, 선택한 기록, 추천 버전을 계획 params나 REF5 런타임 상태에 저장하지 않는다. `startingValuesKg`만 서버가 다시 검증해 저장한다.
- 시작 뒤 모든 진행은 실제 REF5 수행 결과만 사용한다. 기록 갱신으로 최초값을 재계산하거나 기존 계획을 수정하지 않는다.

이 역계수는 기존 REF5 기준점을 일관되게 초기화하기 위한 **프로그램 내부 운영값**이다. 종목별 최적 `%1RM` 또는 모든 훈련자에게 유효한 과학적 시작 강도라고 주장하면 안 된다.

### 5.4 기본값에서의 최초 파생 기준

| 파생 기준 | 시작값 | 처방 |
|---|---:|---:|
| SQ H2 | 87.5 kg | 3×2 |
| SQ V | 72.5 kg | 정상 3×5 / 마이크로 2×5 |
| BP 볼륨 | 70 kg | 1×5 |
| PULL 볼륨 | 총중량 75 kg | 1×6 |

## 6. 중량 계산

### 6.1 반올림 함수

- `floor2.5(x)`: 2.5 kg 격자로 내림한다.
- `nearest2.5(x)`: 가장 가까운 2.5 kg로 반올림하고 정확한 중간값은 위로 올린다.
- PULL 추가중량에는 `max(0, …)`을 먼저 적용한다.

표시용 반올림값을 다시 계산 입력으로 사용하면 안 된다. 직접 기준은 정본이고, 파생 기준·REF·상한은 매번 정본에서 다시 계산한다.

### 6.2 파생 기준 공식

```text
SQ H2                    = SQ H3 + 5
SQ V                     = floor2.5(SQ H3 × 72.5 / 82.5)
BP 볼륨                  = floor2.5(BP 집중 × 70 / 82.5)
PULL 볼륨 목표 총중량   = floor2.5(PULL 집중 목표 총중량 × 75 / 87.5)
```

### 6.3 제어 REF 공식

```text
SQ REF          = 104 × SQ H3 / 82.5
BP REF          = 101 × BP 집중 / 82.5
PULL REF        = 108 × PULL 집중 목표 총중량 / 87.5
DL REF          = 100 × DL 작업중량 / 72.5
OHP REF         = 50 × OHP 작업중량 / 32.5
```

제어 REF는 내부 상태 추적과 보조 상한 계산용이다.

- 1RM, e1RM, TM이 아니다.
- REF÷체중은 장기 추세를 보는 대리지표일 뿐 실제 최대중량을 증명하지 않는다.
- REF 비율을 `%1RM`으로 설명하면 안 된다.

### 6.4 보조 상한

```text
DL 후보 허용  ⇔ 후보 DL REF ≤ 현재 SQ REF
OHP 후보 허용 ⇔ 후보 OHP REF ≤ 현재 BP REF × 0.5
```

작업중량으로 표시한 현재 상한은 다음과 같다.

```text
DL 최대 작업중량  = floor2.5(SQ REF × 72.5 / 100)
OHP 최대 작업중량 = floor2.5((BP REF × 0.5) × 32.5 / 50)
```

기본 시작값의 최초 상한은 DL `75 kg`, OHP `32.5 kg`다. 따라서 기본값으로 시작한 OHP의 최초 4회 창을 모두 PASS해도 `35 kg` 후보는 거부된다. 사용자 지정 시작값에서는 같은 공식으로 상한을 다시 계산한다.

## 7. 세션 생성 순서

세션 판단 순서는 고정이다.

1. 시작 입력을 검증한다.
2. 정상 또는 마이크로를 먼저 결정한다.
3. 현재 PULL↔BP 집중 큐를 읽는다.
4. 세션형과 하드 밀도로 SQ H3/H2/V를 정한다.
5. 직접·파생 기준, REF, 상한, PULL 잠금을 계산한다.
6. 종목 순서·세트·반복·중량을 스냅샷으로 동결한다.
7. SQ 첫 작업세트 시작 때만 START 효과를 확정한다.

세션 중에는 큐·중량·세트·반복을 다시 계산하지 않는다. 해당 세션 결과로 생긴 변화는 다음에 생성되는 세션부터 적용한다.

### 7.1 시작 입력

사용자가 제공하는 값은 다음뿐이다.

- 실제 시작 시각
- 오늘 체중
- 시간 제약에 따른 직접 마이크로 여부

계획의 IANA 시간대는 계획 설정에서 읽는다. 브라우저나 기기의 표시 시간대로 바꾸지 않는다.

최근 7일 PULL 계산 체중, 세션형, 집중 차례, SQ 처방, 중량, 결과 등급, 진행 변화는 앱이 계산한다.

### 7.2 정상 PULL 집중 세션

| 순서 | 처방 | 작업세트 |
|---:|---|---:|
| 1 | SQ H3/H2/V 중 선택 | 3세트 |
| 2 | PULL 집중 | 3×3 |
| 3 | BP 볼륨 | 1×5 |
| 4 | DL | 2×4 |

총 9 작업세트다.

### 7.3 정상 BP 집중 세션

| 순서 | 처방 | 작업세트 |
|---:|---|---:|
| 1 | SQ H3/H2/V 중 선택 | 3세트 |
| 2 | BP 집중 | 3×3 |
| 3 | PULL 볼륨 | 1×6 |
| 4 | OHP | 2×6 |

총 9 작업세트다.

### 7.4 마이크로 세션

| 순서 | 처방 | 작업세트 |
|---:|---|---:|
| 1 | SQ V | 2×5 |
| 2 | BP 볼륨 | 1×5 |
| 3 | PULL 볼륨 | 1×6 |

총 4 작업세트다. 마이크로는 항상 이 형태이며 종목을 자동 생략하거나 대체하지 않는다.

## 8. 정상/마이크로 판단

다음 중 하나라도 참이면 세션은 마이크로다.

1. 사용자가 시간 제약으로 직접 마이크로를 선택함
2. 직전 두 달력일 모두 프로그램일임
3. 직전 여섯 달력일에 정상 세션 시작이 이미 3회 이상임
4. 최근 완료 세션 2회 안에서 서로 다른 주종목 2개 이상의 새 FAIL로 강제 마이크로가 대기 중임
5. SQ, BP, PULL 중 하나 이상의 정체 마이크로가 대기 중임

그 외에는 정상이다. 여러 사유가 겹쳐도 표준 마이크로 한 번만 생성하며 모든 사유를 기록한다.

### 8.1 달력과 시각의 구분

- 달력 규칙은 계획 시간대의 날짜를 사용한다.
- 48시간·168시간 규칙은 실제 시각 사이의 경과시간을 사용한다.
- 프로그램일과 정상 세션 수는 미리보기·예약이 아니라 SQ 첫 작업세트 START로 기록된다.
- 일광절약시간이 있는 시간대에서도 달력일과 경과시간을 섞지 않는다.

### 8.2 강제 마이크로 사건

- 주종목 FAIL마다 안정적인 사건 ID를 만든다.
- 최근 완료 세션 2회 안의 아직 소비되지 않은 사건만 본다.
- 서로 다른 주종목이 2개 이상이면 강제 마이크로 토큰 하나를 만든다.
- 토큰에 사용한 사건은 `CLAIMED`, 최근 2세션에서 벗어난 미사용 사건은 `EXPIRED`가 된다.
- 다음 실제 세션의 SQ 첫 작업세트에서 토큰을 한 번만 소비한다.
- 다른 마이크로 사유가 함께 있어도 이 토큰은 같은 세션에서 소비된다.

### 8.3 정체 마이크로 사건

- 여러 종목의 정체 마이크로와 다른 마이크로 사유는 하나로 합친다.
- 미리보기나 시작 전 취소는 대기를 소비하지 않는다.
- 실제 SQ 첫 작업세트를 시작하면 해당 종목은 재평가 상태로 전환된다.
- 마이크로의 실제 볼륨 결과는 볼륨 FAIL veto와 2연속 FAIL 흐름에는 반영된다.
- 마이크로는 SQ 하드·BP/PULL 집중 노출로 세지 않는다.

## 9. SQ H3/H2/V 선택

정상 세션에서 다음 두 조건을 모두 만족하면 하드 SQ를 사용한다.

1. 마지막 하드 SQ 시작 후 **48시간 이상** 경과
2. 현재 시각 직전 **열린 168시간 구간** 안의 하드 시작이 2회 미만

조건을 만족하면 `H3 ↔ H2` 차례를 사용하고, 만족하지 않으면 `V 3×5`다. 최초 하드는 H3다. 마이크로는 항상 `V 2×5`다.

경계는 정확하다.

- 정확히 48시간: 하드 허용
- 48시간보다 1 ms 짧음: V
- 정확히 168시간 전 시작: 현재 168시간 창에서 제외
- 168시간보다 1 ms 최근: 창 안에 포함

허용오차를 넣는 것은 단순 UI 수정이 아니라 처방 결과를 바꾸는 프로토콜 변경이다. UI는 남은 시간을 친절하게 표시할 수 있지만 엔진 경계를 바꾸면 안 된다.

하드 밀도는 첫 SQ 작업세트 START 때 소비된다. 이후 SQ가 안전·외부 사유로 INVALID여도 해당 하드 시작은 48/168시간 밀도에 남는다. 다만 H3/H2 차례는 SQ 결과가 유효할 때만 교대하므로 INVALID면 같은 하드 차례를 유지한다.

## 10. PULL 총중량과 잠금

PULL의 직접·파생 기준은 추가중량이 아니라 **체중 + 추가중량인 목표 총중량**이다.

### 10.1 계산 체중

오늘 체중은 0.01 kg 단위로 정규화한다. 같은 REF5 plan에서 실제 시작 시각 기준 직전 7×24시간부터 현재까지 동결된 세션 시작 체중과 오늘 입력을 사용한다.

```text
측정 수 ≥ 3: 계산 체중 = 사용 가능한 측정의 산술평균
측정 수 < 3: 계산 체중 = 오늘 체중
```

측정이 2개일 때 두 값의 평균을 쓰면 안 된다. 2개 이하에서는 오늘 값만 쓴다.

### 10.2 추가중량

```text
집중 추가중량 = nearest2.5(max(0, 집중 목표 총중량 - 계산 체중))
볼륨 추가중량 = nearest2.5(max(0, 볼륨 목표 총중량 - 계산 체중))
당일 실제 총중량 = 오늘 체중 + 잠긴 추가중량
```

예: 계산 체중과 오늘 체중이 모두 `75 kg`이면 최초 집중은 `+12.5 kg`, 실제 총중량 `87.5 kg`; 최초 볼륨은 `+0 kg`, 실제 총중량 `75 kg`다.

### 10.3 잠금 수명

- 집중·볼륨 추가중량을 같은 PULL 4회 집중창 동안 함께 잠근다.
- 매일 체중이 바뀌어도 잠긴 추가중량은 유지되고 당일 실제 총중량만 바뀐다.
- PULL 집중 4회 창이 끝나면 그 창을 닫은 세션의 계산 체중으로 다음 창의 잠금을 만든다.
- PULL 직접 목표가 증감하면 즉시 새 목표로 잠금을 다시 만든다.
- 단순 재잠금에서 실제 추가중량이 바뀐 흐름만 연속 FAIL 상태를 초기화하며 정체 상태는 유지한다.
- 직접 목표가 바뀌면 PULL의 창, 두 FAIL 흐름, 활성 정체 단계를 초기화하고 다시 잠근다. 정체 감소 이력과 구조 검토 플래그는 보존한다.

일일 체중 변화와 그에 따른 실제 총중량 변화는 직접 기준 변경이 아니다.

## 11. 세트 종료와 결과 판정

### 11.1 수행 규칙

- 다음 반복에서 정해진 ROM·기술을 유지할 수 없다고 판단되면 그 반복 전에 멈춘다.
- 첫 **명백한 감속** 반복은 유효 반복으로 세고 그 뒤 종료한다.
- 감속인지 애매하면 감속으로 기록하지 않는다.
- 예정 반복을 넘는 추가 반복은 하지 않는다.
- 의도적으로 실패까지 수행하지 않는다.

이 규칙은 속도 기반 처방이나 폭발 수행 지시가 아니다. 수치 속도, RPE, RIR을 입력하지 않는다.

### 11.2 종료 사유

| 원시 사유 | 의미 |
|---|---|
| `NORMAL` | 예정 반복을 명백한 감속 없이 완료 |
| `CLEAR_SLOWDOWN` | 첫 명백한 감속 반복까지 유효 처리하고 종료 |
| `FORCE_OR_TECHNIQUE` | 힘 또는 기술 유지 문제로 남은 반복을 수행하지 않음 |
| `SAFETY` | 안전 사유로 비교 불가 |
| `EXTERNAL` | 장비·시간·환경 등 외부 사유로 비교 불가 |

### 11.3 자동 결과표

예정 반복과 유효 반복의 차이를 모든 세트에서 합한 값을 `총 부족 반복`으로 둔다.

| 결과 | 조건 |
|---|---|
| `PASS` | 총 부족 0, 종료 사유 `NORMAL` |
| `HOLD` | 총 부족 정확히 1, 또는 총 부족 0이지만 `CLEAR_SLOWDOWN` |
| `FAIL` | 비교 가능한 기록에서 총 부족 2 이상 |
| `INVALID` | `SAFETY` 또는 `EXTERNAL` |

추가 제약:

- `NORMAL`인데 부족 반복이 있으면 모순 입력이라 거부한다.
- `FORCE_OR_TECHNIQUE`인데 부족 반복이 0이면 모순 입력이라 거부한다.
- 유효 반복은 예정 반복보다 클 수 없다.
- 세트 수와 예정 반복은 동결 스냅샷과 정확히 일치해야 한다.
- 예정된 모든 종목 결과가 있어야 세션을 닫을 수 있다. 미수행 종목은 실제 사유에 따라 `SAFETY` 또는 `EXTERNAL`로 닫는다.
- 사용자가 결과 등급을 임의 선택하지 않는다. 앱이 원시 기록에서 계산한다.

### 11.4 INVALID의 효과

INVALID는 진행·감소·강제 마이크로용 FAIL로 사용하지 않는다.

- 판정창 노출 수에 넣지 않는다.
- 연속 FAIL을 늘리거나 끊지 않고 건너뛴다.
- 정상 집중 종목이 INVALID면 집중 큐를 유지한다.
- SQ 하드가 INVALID면 하드 시작 밀도는 남지만 H3/H2 차례는 유지한다.
- 같은 세션의 다른 유효 종목 결과는 정상 반영한다.

## 12. 집중 큐

- 집중 큐는 PULL과 BP만 가진다.
- 최초 차례는 PULL이다.
- 정상 세션의 집중 결과가 PASS/HOLD/FAIL이면 세션 종료 후 반대 종목으로 교대한다.
- FAIL도 유효 결과이므로 교대한다.
- 집중 결과가 INVALID면 교대하지 않는다.
- 마이크로는 큐를 소비하지 않는다.
- 결석·휴식일은 큐를 바꾸지 않는다.

## 13. 진행 판정창

### 13.1 SQ

SQ 창에는 정상 세션의 유효 H3/H2만 들어간다. 6회가 모이면 판정한다.

```text
FAIL 0회
AND HOLD 1회 이하
AND 해당 창이 열려 있는 동안 SQ V FAIL 0회
→ SQ H3 직접 기준 +2.5 kg

그 외
→ 유지
```

정상·마이크로 SQ V는 하드 6회에 포함하지 않지만, FAIL이면 다음 SQ 창 증량을 veto한다. 판정 후 하드 노출과 V FAIL veto를 함께 비운다.

### 13.2 BP와 PULL

각 종목의 정상 집중 유효 노출 4회가 모이면 별도로 판정한다.

```text
FAIL 0회
AND HOLD 1회 이하
AND 해당 창이 열려 있는 동안 같은 종목 볼륨 FAIL 0회
→ 직접 집중 기준 +2.5 kg

그 외
→ 유지
```

볼륨 결과는 집중 4회에 포함하지 않는다. 볼륨 FAIL은 다음 집중창 증량만 veto하며 판정 뒤 함께 비운다.

### 13.3 DL과 OHP

정상 세션의 각 보조종목 유효 노출 4회를 모은다.

- 4회가 모두 PASS이고 `현재 +2.5 kg` 후보가 새 주종목 상한 안에 완전히 들어오면 +2.5 kg
- 하나라도 HOLD/FAIL이 있거나 후보가 상한을 넘으면 유지
- 판정 후 4회 창을 비운다.
- DL/OHP에는 주종목식 정체 마이크로·재평가 규칙이 없다.

## 14. 즉시 감소

FAIL은 다음 독립 흐름별로 추적한다.

```text
SQ_H3, SQ_H2, SQ_V_NORMAL, SQ_V_MICRO
BP_FOCUS, BP_VOLUME
PULL_FOCUS, PULL_VOLUME
DL, OHP
```

같은 흐름에서 유효 FAIL이 2회 연속 발생하면 해당 종목의 직접 기준을 다음 등장 전에 `−2.5 kg` 한다.

- 다른 흐름의 결과는 이 연속성을 끊지 않는다.
- 같은 흐름의 PASS/HOLD는 끊는다.
- INVALID는 늘리거나 끊지 않고 건너뛴다.
- 여러 흐름이 같은 종목 감소를 동시에 요구해도 직접 기준은 한 번만 −2.5 kg 한다.
- 즉시 감소와 정체 감소가 겹쳐도 실제 감소는 한 번만 −2.5 kg 한다. 두 원인은 모두 사건으로 남긴다.
- 감소가 증량 판정과 겹치면 감소가 우선한다.

주종목 직접 기준이 바뀌면 해당 종목의 판정창, 볼륨 veto, 모든 FAIL 흐름, 정체 진행을 초기화한다. 집중 큐, SQ H3/H2 차례, 이미 소비한 하드 밀도는 유지한다.

DL/OHP는 자기 감소 후보를 먼저 만들고 새 주종목 상한을 적용한다. 상한이 더 낮으면 2.5 kg 단위로 필요한 만큼 추가 하향하되 자기 감소와 첫 상한 하향을 중복 계산하지 않는다.

## 15. 정체 처리

여기서 `유지창`은 개별 HOLD가 아니라 6회 또는 4회 판정창의 최종 결과가 유지라는 뜻이다.

같은 직접 기준에서의 상태 흐름은 다음과 같다.

```text
첫 유지창
→ 유지창 1회 기록

두 번째 유지창
→ 다음 실제 세션에 정체 마이크로 예약

정체 마이크로 START
→ 재평가 상태

다음 같은 종목 판정창 증가
→ +2.5 kg, 새 기준으로 초기화

다음 같은 종목 판정창 유지
→ 직접 기준 −2.5 kg, 새 기준으로 초기화
```

- 정체 마이크로 뒤 재평가가 끝날 때까지 유효 노출을 새 창에 쌓는다.
- 그 전에 같은 흐름 2연속 FAIL로 즉시 감소하면 감소를 먼저 적용하고 재평가 상태를 초기화한다.
- 정체 감소 이력은 직접 기준 변경 뒤에도 보존한다.
- 동일한 직접 기준값에서 정체 감소가 두 번째 기록되면 감소와 함께 `structureReview`를 켠다. 추가 중복 감소는 하지 않는다.
- `structureReview`는 자동으로 처방을 바꾸는 상태가 아니라 프로그램 구조 재검토 신호다.

## 16. 변화 적용 순서

한 완료 사건에서 여러 조건이 겹칠 수 있으므로 순서를 고정한다.

1. 모든 예정 종목의 원시 결과를 검증·분류한다.
2. 유효 정상 집중이면 PULL↔BP 큐를 교대한다.
3. 유효 SQ 하드면 H3↔H2 차례를 교대한다.
4. 각 흐름의 연속 FAIL과 판정창을 갱신한다.
5. 주종목은 `즉시 감소 → 정체 감소 → 증량 → 유지` 우선순위로 직접 기준을 확정한다.
6. 변경된 주종목 기준으로 파생 기준·제어 REF·보조 상한을 다시 계산한다.
7. DL/OHP 자기 후보 `−2.5 / 0 / +2.5`를 만들고 새 상한을 강제한다.
8. PULL 직접 목표 변경 또는 4회 집중창 종료에 따라 잠금을 다시 만든다.
9. 주종목 FAIL 사건을 저장하고 강제 마이크로 대기를 계산한다.
10. 완료 사건과 모든 변화를 한 번만 기록한다.

같은 2.5 kg 감소 원인을 두 번 적용하거나, 현재 상한 아래의 여유를 다음 증량에 적립하면 안 된다.

## 17. 세션 수명주기와 replay

### 17.1 미리보기

- 현재 과거 상태에서 처방을 계산하지만 쓰기 작업을 하지 않는다.
- 미리보기·취소는 프로그램일, 하드 밀도, 큐, 마이크로 사건, PULL 잠금을 소비하지 않는다.

### 17.2 실제 START

SQ 첫 작업세트를 시작할 때 다음을 원자적으로 확정한다.

- 실제 시작 시각과 계획 달력일
- 정상/마이크로, SQ 처방, 집중 큐
- 모든 중량·세트·반복과 PULL 메타데이터
- 하드 시작 밀도
- PULL 잠금
- 강제·정체 마이크로 대기 소비

같은 `startEventId` 재시도는 한 번만 적용되어야 한다. 같은 이벤트 ID를 다른 세션에 재사용하면 거부한다.

### 17.3 완료

- START와 일치하는 동결 스냅샷이 있어야 한다.
- 예정된 모든 종목 결과가 있어야 한다.
- 완료 시각은 시작 시각보다 이를 수 없다.
- 같은 `completionEventId`와 로그 재시도는 한 번만 적용한다.
- 이미 다른 직접 기준으로 상태가 진행된 동시 저장 충돌은 조용히 덮지 않고 replay를 요구한다.

### 17.4 수정·삭제·과거 삽입

결과 수정, 삭제, 과거 기록 삽입 뒤에는 남은 원시 기록을 처음부터 재생한다.

- 정렬: 실제 시작 시각 → 안정적인 세션 키 → 완료 사건 키
- 중복 이벤트: 동일 의미면 한 번만 사용, 내용이 충돌하면 거부
- 삭제 표시된 로그: 결과에서 제외
- 과거 세션의 동결 처방과 PULL 메타데이터: 보존
- replay 결과: 같은 입력이면 항상 같은 상태

시간이 같은 세션도 안정 키 순서가 있어야 하며, 이미 동결된 같은 시각 묶음 앞에 새 키를 끼워 넣어 과거 판단을 바꾸면 안 된다.

## 18. 앱이 반드시 보여 줄 정보

REF5는 자동화되어 있어도 사용자가 현재 상태를 인지할 수 있어야 한다. 웹과 TUI는 의미가 같은 정보를 제공한다.

- 다음 집중: PULL 또는 BP
- 다음 하드: H3 또는 H2
- 마이크로 대기 여부와 이유
- 오늘 세션이 정상/마이크로인 이유
- SQ H3/H2/V와 `하드`의 뜻
- PULL/BP `집중`과 볼륨의 뜻
- SQ 하드 `현재/6`, BP·PULL 집중 `현재/4`, DL·OHP `현재/4`
- 각 주종목 창의 볼륨 FAIL veto 수
- 직접 기준, 파생 기준, 제어 REF, 보조 상한
- PULL 목표 총중량, 계산 체중, 잠긴 추가중량, 당일 실제 총중량
- 구조 재검토 플래그와 최근 진행 변화

유한 cycle 진행률이나 week/day 진행률로 표시하면 안 된다. 현재/임계값은 **증량 판정까지 모인 유효 노출 수**이지 전체 프로그램 완료율이 아니다.

## 19. 수행 운영 지침

### 19.1 휴식

| 처방 | 권장 휴식 |
|---|---:|
| SQ H3/H2, BP·PULL 집중 | 3–5분 |
| SQ V, BP·PULL 볼륨, OHP | 2–3분 |
| DL | 2–4분 |
| 마이크로 | 2–3분 |

워밍업은 작업세트 수에 포함하지 않는다. 특정 워밍업 프로토콜을 엔진이 강제하지 않지만, 비교 가능성을 위해 일관된 방식이 권장된다.

### 19.2 결석과 중단

- 놓친 세션을 보충하지 않는다.
- 다음 실제 세션에서 현재 상태와 실제 시작 시각으로 다시 계산한다.
- 시작 후 일부 종목을 수행하지 못했으면 해당 종목을 실제 안전/외부 사유로 INVALID 처리하고 세션을 닫는다.
- 세션 중 처방을 다른 종목으로 대체하지 않는다.

### 19.3 예상 작업세트

일정이 장기적으로 분산되고 집중 큐가 교대한다는 가정의 근사치다.

| 실제 빈도 | SQ | BP | PULL | DL | OHP |
|---:|---:|---:|---:|---:|---:|
| 주 2회 | 6 | 4 | 4 | 2 | 2 |
| 주 3회 | 9 | 약 6 | 약 6 | 약 3 | 약 3 |
| 주 4회 | 약 11 | 약 7 | 약 7 | 약 3 | 약 3 |

달력 배치와 마이크로 사유에 따라 달라지며 보장값이 아니다. 예를 들어 3일 연속이면 정상 2회+마이크로 1회로 SQ 8, BP/PULL 각 5, DL/OHP 각 2세트가 될 수 있다.

## 20. 개인 운영 프로필과 프로토콜의 분리

다음은 이 프로그램을 만든 장기 운영 맥락이지만 세션 상태머신 입력은 아니다.

- 체중 약 70 kg과 하이바 SQ 175 kg의 동시 장기 목표
- 의도적 증량기 없음
- 첫 평가 게이트까지 현재 7일 평균 체중을 대체로 유지
- 이후 필요 시 주당 약 7일 평균 체중의 0.25% 수준으로 보수적 감량
- 단백질 섭취 시작 범위 1.6–2.2 g/kg/day
- SQ 정체 재평가 감소나 즉시 감소가 발생하면 감량을 멈추고 회복 상태 검토

앱은 오늘 체중을 PULL 계산에 사용하지만, 70 kg 목표·감량률·단백질 섭취를 세션 처방 분기로 사용하면 안 된다. 개인 목표를 바꾸는 것은 REF5 프로토콜 변경이 아니며, PULL 총중량 계산 규칙을 바꾸는 것은 프로토콜 변경이다.

직접 1RM 검사가 없으므로 제어 REF나 REF÷체중만으로 실제 SQ 175 kg 달성을 확정하면 안 된다.

## 21. 최초 평가 게이트

최소한 다음 유효 노출이 쌓이기 전에는 한두 세션의 느낌만으로 구조를 바꾸지 않는다.

- SQ 하드 6회
- BP 집중 4회
- PULL 집중 4회
- DL 4회
- OHP 4회

다만 명백한 구현 오류, 안전 문제, 상한 위반, 중복 감소 등 반증 사건은 즉시 수정한다.

평가 시 최소 기록:

- 예정 세트 완료율과 PASS/HOLD/FAIL/INVALID 분포
- SQ 직접 기준과 제어 REF
- 제어 REF÷7일 평균 체중 추세(대리지표임을 명시)
- 실제 주 2/3/4회별 종목 작업세트
- 정상·마이크로 세션 시간
- 즉시 감소, 정체 감소, 구조 재검토 발생 여부

## 22. 근거와 한계

### 22.1 연구가 비교적 직접 지지하는 넓은 방향

- 저항훈련은 근력 향상에 효과가 있고, 우선 종목을 세션 앞에 두며 다세트·규칙적 노출을 사용하는 방향은 근거가 있다. 다만 연구의 부하 기준은 실제 1RM인 경우가 많아 REF5 시작 REF와 동일시할 수 없다. [ACSM 2026 Position Stand](https://pubmed.ncbi.nlm.nih.gov/41843416/), [exercise-order systematic review](https://pubmed.ncbi.nlm.nih.gov/32077380/)
- 빈도와 볼륨은 효과가 있으나 체감효과가 감소하고, 빈도 효과는 볼륨 통제 여부와 대상에 따라 해석이 달라진다. [frequency meta-analysis](https://pubmed.ncbi.nlm.nih.gov/29470825/), [dose-response meta-regressions](https://pubmed.ncbi.nlm.nih.gov/41343037/)
- 근력 향상을 위해 매 세트를 실패까지 수행할 필요는 없다. [failure vs non-failure meta-analysis](https://pubmed.ncbi.nlm.nih.gov/33497853/), [proximity-to-failure meta-regressions](https://pubmed.ncbi.nlm.nih.gov/38970765/)
- 훈련 경험자의 근력 향상에는 2분을 넘는 비교적 긴 세트 간 휴식이 유리할 수 있다. [rest-interval systematic review](https://pubmed.ncbi.nlm.nih.gov/28933024/)

### 22.2 넓은 방향만 참고한 근거

- 1RM 추정은 낮은 반복수 기록일수록 오차가 줄어드는 경향이 있어 시작 추천은 10회 이하로 제한한다. 다만 반복수–%1RM 관계에는 종목·개인차가 크므로 e1RM과 역계수 결과는 첫 처방 후보일 뿐 실제 능력이나 최적 부하의 증명이 아니다. [Reynolds et al. 2006](https://pubmed.ncbi.nlm.nih.gov/16937972/), [Nuzzo et al. 2024](https://pubmed.ncbi.nlm.nih.gov/37792272/)
- 자동조절 연구는 APRE·RPE·속도 기반 방식 등을 다룬다. REF5의 반복 부족 기반 판정기를 직접 검증하지 않는다. [autoregulation review](https://pubmed.ncbi.nlm.nih.gov/40791980/)
- 유연한 일정과 변동 처방 연구는 세션 유연성·부하 변동의 가능성을 지지하지만 PULL↔BP 큐나 H3/H2/V 선택기를 검증하지 않는다. [flexible scheduling trial](https://pubmed.ncbi.nlm.nih.gov/28129275/), [periodization meta-analysis](https://pubmed.ncbi.nlm.nih.gov/35044672/)
- 예정 중단 주간의 우월성은 확정적이지 않다. REF5가 고정 디로드 대신 반응형 마이크로를 쓰는 것도 직접 비교로 우월성이 입증된 선택은 아니다. [one-week cessation trial](https://pubmed.ncbi.nlm.nih.gov/38274324/), [deload practices survey](https://pubmed.ncbi.nlm.nih.gov/38499934/)
- 단백질·에너지 부족 문헌은 개인 운영 프로필의 보수적 시작 범위를 정하는 참고다. 정확한 개인 최적값을 보장하지 않는다. [protein meta-analysis](https://pubmed.ncbi.nlm.nih.gov/28698222/), [energy-deficiency meta-analysis](https://pubmed.ncbi.nlm.nih.gov/34623696/)

### 22.3 직접 검증되지 않은 REF5 운영 가설

다음 숫자와 상태 전이는 연구가 직접 최적화한 값이 아니다.

- 48시간·168시간 SQ 하드 경계
- 직전 이틀, 이전 6일 정상 3회 마이크로 경계
- 최근 2세션의 서로 다른 주종목 FAIL
- SQ 6회, BP/PULL/DL/OHP 4회 판정창
- HOLD 1회 허용과 볼륨 FAIL veto
- 같은 흐름 2연속 FAIL 즉시 감소
- 유지창 2회 → 마이크로 → 재평가 유지 시 감소
- PULL 4회 집중창 추가중량 잠금
- 모든 직접 기준의 ±2.5 kg 조절
- e1RM에서 최초 직접 기준으로 바꾸는 다섯 역계수와 최근 56일 선택창

이 값들은 비교 가능성, 낮은 입력 부담, 불규칙 일정 대응, 과도한 상태 증가 방지를 위한 설계 가설이다. “과학적으로 검증된 최적 수치”라고 표현하면 안 된다.

### 22.4 알려진 취약점

1. 주 2회에서는 BP·PULL 각 4세트가 숙련자에게 부족할 수 있다.
2. 주 4회 마이크로 제한은 회복력이 높은 경우 과소자극일 수 있다.
3. 정체 감소는 과중량과 저볼륨을 구분하지 못한다.
4. 주 2–3회에서는 정체 재평가까지 달력 시간이 길다.
5. OHP +2.5 kg는 시작 중량 대비 큰 단계이고 상한 때문에 장기 유지될 수 있다.
6. PULL 잠금은 창 안 비교성을 높이지만 일일 체중만큼 실제 총중량이 변한다.
7. 명백한 감속의 이진 판정은 장비 부담이 낮은 대신 수치 속도·당일 여유 정보를 버린다.
8. 직접 최대중량 검사가 없어 장기 목표 달성 여부를 제어 REF만으로 증명할 수 없다.

## 23. 반증·구조 재검토 조건

다음은 일시적 불편이 아니라 프로그램 또는 구현을 재검토할 근거다.

- 보조 REF 상한이 한 번이라도 깨짐
- 정체 마이크로가 다음 실제 세션에 생성되지 않음
- 즉시 감소와 정체 감소가 실제 중량에 중복 적용됨
- 같은 이벤트 재시도로 큐·창·중량이 두 번 변함
- 수정·삭제 후 replay가 같은 원시 기록에서 다른 결과를 만듦
- 정상 9세트를 실제 시간 제약에서 지속적으로 수행할 수 없음
- 주 2회에서 BP·PULL이 반복적으로 유지조차 못 하거나 명백히 자극이 부족함
- 주 4회 마이크로 뒤에도 서로 다른 주종목 FAIL이 반복됨
- 정체 재평가 1주기와 재구축 뒤에도 기준·유효 완료율이 개선되지 않음
- 동일 직접 기준값에서 두 번째 정체 감소가 발생함
- 체중 변화와 작업 기준 하락이 반복적으로 함께 나타남

한두 세션의 변동만으로 구조 우열을 결론내리지 않는다.

도메인 스냅샷은 시작 입력, 세션 판단, 모든 기준·REF·상한, PULL 계산 맥락, 종목별 stream·세트·반복·중량을 보존한다.

### 24.3 버전 경계

- 활성 v1.2 쓰기는 엔진 `512`, 프로토콜 `1.2`, 런타임·도메인 스냅샷 스키마 `2`를 사용한다.
- 새 계획의 사용자 지정 직접 기준은 시작 구성 `1`로 정본화한다. 이는 기존 `startingValuesKg` 상태 모양을 사용하며 세션 전이·공식·decoder를 바꾸지 않으므로 검토 결과 런타임 프로토콜은 `1.2`를 유지한다.
- 일회성 e1RM 시작 도우미는 계획 생성 전 UI/API 계산이다. 저장되는 `startingValuesKg`의 의미와 모양을 바꾸지 않으므로 시작 구성과 런타임 프로토콜을 올리지 않는다.
- 기존 계획의 시작 구성은 불변이다. 새 시작값을 기존 replay에 주입하거나 과거 스냅샷을 다시 쓰지 않는다.
- `511`은 v1.1 식별자이며 v1.2로 재해석하지 않는다.
- 현재 v1.2 decoder는 protocol-less, v1.1, 삭제된 외부 활동 입력을 거부한다.
- seed에 남은 v1.1 버전 메타데이터는 legacy 보존용이며 활성 v1.2 로직과 섞지 않는다.
- 향후 실제 legacy 기록을 지원해야 한다면 명시적 decoder/reducer와 전환 사건을 추가한다. v1.2로 소급 판독하지 않는다.

## 27. 기준 예시

체중과 7일 계산 체중이 모두 75 kg이고 최초 상태라고 가정한다.

### 27.1 첫 정상 세션

| 순서 | 처방 |
|---:|---|
| 1 | SQ H3 82.5 kg, 3×3 |
| 2 | PULL 총 87.5 kg = 체중 75 + 추가 12.5 kg, 3×3 |
| 3 | BP 70 kg, 1×5 |
| 4 | DL 72.5 kg, 2×4 |

정상·PULL 집중·하드 H3이며 총 9세트다. 유효 완료 뒤 다음 집중은 BP, 다음 하드는 H2다.

### 27.2 다음 BP 집중 세션

하드 시간·밀도 조건을 만족한다고 가정한다.

| 순서 | 처방 |
|---:|---|
| 1 | SQ H2 87.5 kg, 3×2 |
| 2 | BP 82.5 kg, 3×3 |
| 3 | PULL 총 75 kg = 체중 75 + 추가 0 kg, 1×6 |
| 4 | OHP 32.5 kg, 2×6 |

총 9세트다. 하드 조건을 만족하지 않으면 첫 처방만 SQ V 72.5 kg, 3×5로 바뀐다.

### 27.3 마이크로

| 순서 | 처방 |
|---:|---|
| 1 | SQ V 72.5 kg, 2×5 |
| 2 | BP 70 kg, 1×5 |
| 3 | PULL 총 75 kg, 1×6 |

총 4세트다. 집중 큐와 하드·집중 판정창 노출 수는 그대로다.

### 27.4 결과 예시

3×3 처방 기준:

- `3/3/3`, 정상 종료 → PASS
- `3/3/3`, 마지막 유효 반복에서 명백한 감속 → HOLD
- `3/3/2`, 힘·기술 사유 → HOLD
- `3/2/2`, 힘·기술 사유 → FAIL
- 안전 또는 외부 사유 → INVALID

### 27.5 진행 예시

- PULL 집중 4회가 `PASS/HOLD/PASS/PASS`이고 PULL 볼륨 FAIL이 없으면 목표 총중량 `87.5→90 kg`이다.
- PULL 집중 흐름에서 FAIL이 두 번 연속이면 4회 창 완료 전이라도 목표 총중량을 `−2.5 kg` 한다.
- OHP가 4회 모두 PASS여도 `35 kg`가 BP 기반 상한을 넘으면 `32.5 kg`를 유지한다.
- SQ가 같은 직접 기준에서 두 판정창 연속 유지되면 다음 실제 세션은 마이크로다. 그 뒤 재평가 창도 유지되면 SQ H3를 `−2.5 kg` 한다.
